# CareCompanion App Audit V2 — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 21 audit issues and ship 6 scope expansions: smart care timeline, guided onboarding, caregiver/patient mode, social auth, treatment cycle tracking, and emergency lock screen widget.

**Architecture:** Unified API client across all mobile tabs. Server-side /api/timeline endpoint using UNION ALL query. Onboarding state computed from existing data. New treatment_cycles table for cycle tracking. iOS WidgetKit for emergency card. Apple + Google Sign-In via Cognito.

**Tech Stack:** React Native/Expo (mobile), Next.js 14 (web), Drizzle ORM, AWS Cognito, AWS Aurora PostgreSQL, Vercel AI SDK, WidgetKit (iOS)

**Reviews Completed:** CEO (SCOPE_EXPANSION, 6/6 accepted), Eng (CLEAR), Design (4→8/10)

## Chunk 3: Phase 3 — Smart Care Timeline (Expansion #2) + Onboarding (Expansion #1)

### Task 8: Create /api/timeline Endpoint

**Files:**
- Create: `apps/web/src/app/api/timeline/route.ts`
- Modify: `apps/web/src/lib/db/schema.ts` (verify table references)

- [ ] **Step 1: Create the timeline API endpoint**

`GET /api/timeline?profileId=X&days=7`

Query medications, appointments, and labs for the given profile and date range. Use UNION ALL to merge into a single sorted result:

```sql
SELECT 'medication' as type, name as title, dosage as subtitle, scheduled_time as timestamp, id
FROM medication_reminders WHERE care_profile_id = ? AND scheduled_time BETWEEN ? AND ?
UNION ALL
SELECT 'appointment' as type, title, doctor_name as subtitle, appointment_time as timestamp, id
FROM appointments WHERE care_profile_id = ? AND appointment_time BETWEEN ? AND ?
UNION ALL
SELECT 'lab' as type, test_name as title, 'Due' as subtitle, due_date as timestamp, id
FROM lab_results WHERE care_profile_id = ? AND due_date BETWEEN ? AND ?
ORDER BY timestamp ASC
```

- [ ] **Step 2: Add medication refill predictions**

- [ ] **Step 1: Add treatment_cycles to schema**

```typescript
export const treatmentCycles = pgTable('treatment_cycles', {
  id: uuid('id').primaryKey().defaultRandom(),
  careProfileId: uuid('care_profile_id').notNull().references(() => careProfiles.id),
  cycleNumber: integer('cycle_number').notNull(),
  startDate: date('start_date').notNull(),
  cycleLengthDays: integer('cycle_length_days').notNull().default(21),
  regimenName: text('regimen_name'),
  notes: text('notes'),
  isActive: boolean('is_active').notNull().default(true),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
});
```
