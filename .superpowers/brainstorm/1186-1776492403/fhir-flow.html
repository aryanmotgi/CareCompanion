Manual FHIR Export — How It Would Work

User journey from onboarding → fully populated profile

The Flow

1

User signs up → hits onboarding

Instead of the current wizard form, we show: "Import your health records in one step"

2

We ask: "What patient portal does your doctor use?"

User picks from a list: Epic MyChart · CommonHealth · Cerner · Kaiser · UCLA Health · etc.

3

CareCompanion shows step-by-step instructions

Tailored to their portal — with screenshots. e.g. for MyChart: "Tap Menu → Health → Download My Data → Select All → Download"

4

User uploads the file (JSON or XML)

Drag & drop or file picker. Usually a single .json file or a zip.

5

CareCompanion auto-fills everything

Medications · Conditions · Allergies · Lab results · Doctors · Diagnoses — all imported. User just reviews and confirms.

What gets imported from one FHIR file

✓ Reliably present
  • Active medications + doses
  • Diagnoses / conditions
  • Allergies
  • Lab results (historical)
  • Immunizations
  • Procedures
  • Care team / providers
~ Sometimes present
  • Upcoming appointments
  • Insurance / coverage info
  • Prior authorizations
  • Pharmacy info
  • Vitals history

Honest tradeoffs

Pros

  • No approval needed — ships now
  • Works with any US hospital
  • One file = entire medical history
  • User controls their data
  • No ongoing API dependency

Cons

  • Not automated — user does 5-10 steps
  • Export UI buried in portals
  • One-time snapshot (not live-syncing)
  • Some users won't have portal access
  • File parsing complexity
